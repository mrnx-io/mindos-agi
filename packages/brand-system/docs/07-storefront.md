<a id="SECTION_STOREFRONT_CONSTRAINTS"></a>
## III. C O N S T R A I N T S

Without constraints, the artifact becomes a different project than the one the system designed. These laws ensure the proof actually proves what six layers built:

**Every decision traces upstream.**
No arbitrary choices. If a color is used, it comes from Sumi Breath tokens. If a word is used, it comes from the Canon lexicon. If an interaction exists, it serves a Moment of Truth.

**The Funnel is the spine.**
The artifact structure follows the Covenant's funnel architecture. Temperature → Pre-frame → Qualify → Sell → Ascend. Every page/section/screen maps to a stage.

**The Value Ladder is the skeleton.**
The artifact doesn't just convert — it ascends. Bait → Frontend → Middle → Backend → Continuity. Each rung must be buildable from the artifact.

**The 10 DR Rules are non-negotiable.**
These rules have been proven across billions of dollars in revenue. Violations predict failure.

**Moments of Truth are sacred.**
The Experience Choreography defines the 5-12 critical moments. These receive maximum attention. Everything else is infrastructure.

**LF8 is the foundation.**
Every section must serve the LF8 hierarchy. Pain-relief first. Ease second. Edge where appropriate. If an element serves no biological driver, it doesn't belong.

**The Slippery Slide is the principle.**
Every element exists to pull the visitor to the next element. Resistance points are conversion leaks.

**Economics are honored.**
The artifact must operate within Unit Economics constraints. No features that break the CAC target. No complexity that extends payback period.

**The Tradeoff Ladder resolves conflicts.**
When implementation forces a choice (speed vs. craft, clarity vs. completeness), consult the Ladder. The higher-ranked principle wins.

**Anti-patterns are boundaries.**
The Experience Choreography's anti-patterns are hard constraints. The artifact must never do what competitors do that violates the Manifesto.

_The constraints are set. Now: the required atoms._

<a id="SECTION_STOREFRONT_THEPARADOXVERIFICATION"></a>
### 10. T H E P A R A D O X V E R I F I C A T I O N

The artifact must pass both tests:

**Logic Test:**

- Every design decision traces to a token or rule
- Every word traces to Lexicon or Voice Rules
- Every interaction traces to a Moment of Truth
- A rationalist finds nothing arbitrary

**Mystery Test:**

- The whole exceeds the explanation
- Something is felt that cannot be fully named
- The visitor experiences transformation, not just transaction
- A poet finds something in the artifact that surprises them

**Mystery Proxy Audit (machine-checkable, not a substitute):**

- **Whitespace ratio** (0–1): proportion of negative space per viewport
- **Information density**: words + interactive elements per viewport
- **Open-loop ratio** (0–1): % of sections that end with a curiosity seed
- **Explanation → implication ratio**: named reasons before felt consequence
- **Named reasons before consequence**: count until first emotional/identity payoff

**If only one test passes, the artifact is incomplete.**
